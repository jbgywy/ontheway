# 需求转化框架

## 用途
将终端用户的隐性需求转化为可操作的产品功能、服务流程或商业模式。

## 适用场景
- 新产品开发
- 现有产品迭代优化
- 服务流程再造
- 商业模式创新

## 核心流程（四步法）

### 步骤1：需求挖掘（Discovery）
**目标：从用户行为中提取真实需求（而非表面诉求）**

**工具1：Jobs-to-be-Done (JTBD) 框架**
```
不要问："你想要什么功能？"
要问："你雇佣这个产品来完成什么任务？"

JTBD模板：
"当我[场景]，我想要[动机]，这样我能[期望结果]。"
```

**工具2：用户旅程地图（User Journey Map）**
```
绘制用户从接触到使用完成的全流程：
1. 认知阶段：如何了解到产品？
2. 考虑阶段：如何评估备选方案？
3. 购买阶段：如何决策？
4. 使用阶段：如何使用？遇到什么痛点？
5. 忠诚阶段：为什么会复购/推荐？
```
标注每个阶段的：
- 用户情绪曲线（高兴😊 → 沮丧😞 → 满意😊）
- 痛点（pain points）
- 机会点（opportunity areas）

**工具3：深度访谈技巧**
- 问"为什么"至少5次（5 Whys）
- 观察用户实际操作（而非听他们说什么）
- 记录用户原话（保留语境和情感）

### 步骤2：需求分类（Categorization）
**目标：将需求按重要性和实现难度分类**

**框架：Kano模型**
| 需求类型 | 说明 | 策略 |
|---------|------|------|
| 基本型（Must-be） | 不满足会不满意，满足不会特别满意 | 必须做好，但不能差异化 |
| 期望型（Performance） | 满足越多越满意 | 核心竞争力，重点投入 |
| 兴奋型（Delighter） | 超出预期，带来惊喜 | 差异化亮点，适度投入 |
| 无差异型（Indifferent） | 有无不影响满意度 | 不投入 |
| 反向型（Reverse） | 满足反而会不满意 | 避免 |

**优先级排序：基本型 > 期望型 > 兴奋型**

**工具：需求优先级矩阵**
```
影响力/重要性
    ^
    |
    |  期望型需求       兴奋型需求
    |  （重点投入）     （差异化亮点）
    |
----+-------------------------→ 实现难度
    |
    |  基本型需求       无差异型需求
    |  （必须做好）     （不投入）
    |
```

### 步骤3：需求转化（Translation）
**目标：将抽象需求转化为具体功能/服务**

**方法1：用户故事（User Story）**
```
模板：
"作为[用户角色]，我想要[功能]，这样我能[价值]。"

示例：
"作为投保人，我想要在线申请理赔，这样我能更快拿到赔款。"
```

**方法2：验收标准（Acceptance Criteria）**
```
Given-When-Then格式：
Given（给定）：初始状态
When（当）：用户执行操作
Then（那么）：系统响应

示例：
Given 用户已登录
When 用户上传理赔材料
Then 系统应在24小时内审核并反馈
```

**方法3：服务蓝图（Service Blueprint）**
```
将服务流程可视化为5层：
1. 用户行为层（用户做什么）
2. 前台接触层（员工/系统做什么）
3. 后台支持层（后台做什么）
4. 支持过程层（系统/流程支持）
5. 证据层（用户看到什么）
```

### 步骤4：需求验证（Validation）
**目标：低成本快速验证需求真实性**

**方法1：MVP（最小可行产品）测试**
- 用最简化的产品测试核心假设
- 衡量指标：转化率、留存率、NPS

**方法2：A/B测试**
- 同时测试两个版本
- 数据驱动决策

**方法3：原型测试**
- 低保真原型（纸面/线框图）：测试概念
- 高保真原型（可交互Demo）：测试体验

## 应用案例
- ✅ 微信支付：从"红包"场景切入，满足"社交+支付"需求
- ✅ 特斯拉OTA升级：将"维修"需求转化为"软件升级"体验
- ✅ 寿险在线理赔：将"跑腿提交材料"需求转化为"拍照上传"体验

## 注意事项
1. **避免"伪需求"**：
   - 用户说想要 ≠ 用户真的会用的
   - 解决方案：观察用户行为，而非听用户说什么
2. **避免"功能蔓延"**：
   - 不是所有需求都要实现
   - 解决方案：严格按Kano模型优先级排序
3. **避免"技术导向"**：
   - 不是所有能做的都要做
   - 解决方案：从用户价值出发，而非技术可行性出发

## 工具推荐

- **用户旅程地图**：Miro、Smaply、UXPressia
- **原型设计**：Figma、Axure、Sketch
- **需求管理**：Jira、ProductPlan、Aha!
